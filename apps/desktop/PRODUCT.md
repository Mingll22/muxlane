# Product

## Register

product

## Platform

web

## Users

Windows 10/11 与 WSL2 上同时维护多个本地项目的 Codex CLI 重度用户。他们需要在桌面工作台中安全管理本人账号、项目运行时、持久终端和恢复事件，同时保持终端工作流的速度与可见性。

## Product Purpose

Muxlane Desktop 是正式 `muxlaned` Runtime Control Plane 的 Windows 操作界面。成功意味着用户无需手工移动凭证或猜测后台状态，即可完成初始化、账号与额度查看、项目注册和启动、终端开发、异常恢复与跨 GUI 会话续接。

## Positioning

把可恢复的 WSL Codex Runtime 变成一个终端优先、事实可追溯且不会接管用户其他 WSL 工作负载的 Windows 工作台。

## Brand Personality

克制、精确、沉着。界面应像可靠的本地控制台，反馈直接但不惊扰，不模仿 OpenAI、Codex 官方或其他同类产品。

## Anti-references

不使用营销型 hero、装饰性玻璃卡片、过度霓虹、连续大卡片网格或把管理指标置于终端之上的仪表盘结构；不复制竞品布局、素材、文案或视觉标识。

## Design Principles

- 终端始终是视觉与交互主体，管理能力按需出现。
- 所有状态都指出事实来源、刷新时间与下一步操作。
- 高风险动作显式、可取消、不会静默切号或关闭 WSL。
- 密度服务于熟练用户，但键盘路径、层级和状态色保持清晰。
- GUI 是可丢弃的客户端；后台 Runtime、事务与 Session 不依赖窗口存活。

## Accessibility & Inclusion

键盘可操作，焦点清晰；正文与状态信息满足 WCAG AA 对比度；状态不只依赖颜色；支持减少动态效果、常见 Windows 缩放和 720px 以上窗口宽度。
